Timing Attack Demonstration

This page shows two vulnerable PIN-entry designs. Demo 1 visibly rejects as soon as a wrong digit is pressed, so the attacker learns the first wrong position instantly. Demo 2 waits for all 4 digits, but the internal comparison still returns early, so a scope trace on the LED line reveals the timing difference.

Demo 1: Immediate feedback on wrong digit

There is no Enter key. Pressing a wrong digit immediately lights the red LED. Correct prefix digits do nothing until all 4 are entered.

What leaks?

Example vulnerable C code

// Immediate feedback version: reject as soon as a wrong digit is pressed.
static const char stored_pin[4] = {'0', '7', '2', '3'};

bool press_digit_early(char digit, unsigned position)
{
    if (digit != stored_pin[position]) {
        led_reject_on();      // Visible leak: wrong position discovered immediately
        clear_entered_pin();
        return false;
    }

    append_digit(digit);

    if (position == 3) {
        led_accept_on();
        clear_entered_pin();
        return true;
    }

    return false;             // Prefix still valid, keep going
}

Why it is still vulnerable

The UI waits for all 4 digits, but internally the PIN check still stops on the first wrong digit. The LED line changes after the internal work is finished, so the pulse width leaks how many leading digits were correct.

Example vulnerable C code

// Full-PIN entry, but still leaks because comparison exits early.
static const char stored_pin[4] = {'0', '7', '2', '3'};

bool check_pin_timed(const char input[4])
{
    for (unsigned i = 0; i < 4; i++) {
        do_some_work();       // Power/timing grows with correct prefix length
        if (input[i] != stored_pin[i]) {
            return false;     // Scope sees the operation end early
        }
    }
    return true;
}

// Constant-time style fix:
bool check_pin_constant(const char input[4])
{
    unsigned diff = 0;
    for (unsigned i = 0; i < 4; i++) {
        diff |= (unsigned)(input[i] ^ stored_pin[i]);
    }
    return diff == 0;
}
